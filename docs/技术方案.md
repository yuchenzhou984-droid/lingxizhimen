# 技术方案

## 技术选型

| 层面 | 选择 | 原因 |
|------|------|------|
| 框架 | 无框架，原生 HTML/CSS/JS | 零依赖，Gitee Pages 直接部署 |
| 样式方案 | 原生 CSS + CSS Variables | 暗色主题切换、响应式 |
| 图标 | SVG 内联 / Emoji | 零外部依赖 |
| 字体 | 系统字体栈 | 不加载外部字体，保证加载速度 |
| 数据存储 | 无（纯前端） | 静态网站，无需持久化 |
| 路由 | 单页应用（Hash 路由） | 多页面体验，无需服务端配置 |

## 架构设计

### 单页应用结构
```
index.html          # 入口，包含所有页面
├── 首页            # 三大模块导航入口
├── 测试一页面      # 14题女巫类型测试
├── 测试二页面      # 7题魔杖定制测试
├── 塔罗占卜页面    # 牌阵选择 → 抽牌 → 解读
├── 壁纸生成页面    # 基于女巫类型的壁纸生成
└── 结果展示页面    # 测试结果复看
```

### 数据流
- 测试答案 → 计分逻辑 → 女巫类型 → 结果展示
- 女巫类型 → 测试二魔杖结果关联
- 女巫类型 → 壁纸生成参数
- 塔罗牌阵选择 → 随机抽牌 → 解读文案匹配

### 关键实现方案

**测试系统**
- 每题选项对应不同维度的分数权重
- 14题结束后，按6个维度的累计分判定女巫类型
- 测试二第1题可手动选择或自动带入测试一结果

**塔罗系统**
- 牌面数据以 JSON 格式存储
- 包含牌名、关键词、正位解读、逆位解读
- 洗牌算法（Fisher-Yates）保证随机性
- 解读文案以模板方式组合，避免重复

**壁纸生成（Canvas）**
- HTML5 Canvas 动态绘制
- 背景层 + 符号元素层 + 文字层
- 使用 `canvas.toBlob()` 或 `canvas.toDataURL()` 导出
- 移动端适配不同屏幕分辨率

**分享功能**
- 使用 Web Share API（移动端主流浏览器支持）
- 降级方案：复制链接 / 保存图片到相册

## 浏览器兼容
- iOS Safari 14+
- Android Chrome 90+
- 桌面 Chrome / Safari / Edge 最新版
- 不兼容 IE
